Set up Zeitspiegel at your studio

Total time: about 30 minutes of computer work, plus shipping time for the parts. The flashing tools run on macOS or Linux; everything else is plug and play.

1. What to buy

Total parts cost is roughly €120–€180, plus a screen.

PartNotes
Raspberry Pi 5 (4 GB is fine; 8 GB if you want long buffers) The brain. Get the 4 GB for normal use; the 8 GB is only worth it if you want delays longer than a minute or two at 1080p.
raspberrypi.com · find a reseller
Official Raspberry Pi 5 power supply (27 W / 5 V 5 A USB-C) Required, not optional — the camera draws power from the Pi and the video encode pushes all four cores. Third-party USB-C bricks tend to be 3 A and will brownout under load.
search
Active cooler for Pi 5 The official one with the small fan is fine. Without it the Pi throttles during MP4 export.
search
micro-SD card, A2 rating, 16 GB or more Any reputable brand (SanDisk, Samsung, Kingston). The whole appliance fits in < 4 GB; the rest is headroom.
Razer Kiyo webcam (USB) This is the recommended camera. It outputs MJPEG natively at 720p60 and has a hardware ring light controlled by a bezel. Other UVC webcams with native MJPEG should work too, but Kiyo is what's actually tested.
search
Micro-HDMI → HDMI cable The Pi 5 has micro-HDMI sockets. One cable is enough.
Any HDMI display Whatever screen you want to use as the mirror. A 32" TV is a good starting size; bigger is better if guests will read it from across the room.
You'll also need: a way to mount the camera near or on the screen, pointed at where dancers will stand. Many studios use a small desktop tripod sitting on top of the TV, or a clamp on a mic stand.

2. What you need on your computer

Flashing the SD card is done from your Mac (or any Linux machine — see the note below). You need:

  • Docker Desktop — the image is baked inside a Linux container, so the recipe is the same on any host. docker.com.
  • Git — usually already installed; if not, install Xcode Command Line Tools (xcode-select --install) or git-scm.com.
  • Go 1.25 or newer — used to cross-build the Pi binary. go.dev/dl.
  • An SSH key at ~/.ssh/id_*.pub. The flasher copies it onto the Pi so you can later ssh in. If you don't have one: ssh-keygen -t ed25519.

Not on macOS? The baking step works anywhere Docker runs. Writing the card is the only Mac-specific bit (make sd); on Linux, after make image, write the resulting build/zeitspiegel-appliance.img with dd or with the Raspberry Pi Imager's "Use custom image" option.

3. Get the code and flash a card

git clone https://github.com/DanielMSchmidt/zeitspiegel.git
cd zeitspiegel

Insert your micro-SD card into your Mac, then:

make sd

That single command downloads Raspberry Pi OS Lite (cached for next time), cross-builds the Zeitspiegel binary, bakes everything into a finished image inside Docker, then asks you to confirm before erasing the card and writing the image.

If you want a different Wi-Fi name than the default zeitspiegel:

SSID=studio-mirror make sd
What gets baked in: the binary, ffmpeg, SDL2, the systemd service, an open Wi-Fi hotspot, your SSH key, and a passwordless sudo user. A random console password is generated and saved to build/credentials.txt in the repo. Keep it; you'll only need it if you ever plug a keyboard into the Pi directly.

4. First boot

  1. Put the card into the Pi.
  2. Plug in the camera, the HDMI cable to your screen, and finally the power.
  3. Wait about three minutes. The Pi resizes its filesystem, starts the Wi-Fi hotspot, locks itself into read-only mode, and reboots a couple of times. No interaction needed.
  4. You're done when the Wi-Fi network zeitspiegel (or whatever name you chose) appears on your phone.

5. Use it

  1. Connect your phone to the Wi-Fi network zeitspiegel. There is no password.
  2. Your phone might warn "this network has no internet" — that's expected and correct. Stay connected.
  3. Open http://zeitspiegel.local in any browser. (If that doesn't resolve, try http://10.42.0.1.)
  4. Use the slider to set the delay. Use the download button to save the last clip to your phone.

Print the guest poster and tape it next to the screen so dancers know what to do.

6. Day-to-day operation

  • To start: plug in power. The full-screen mirror is up in about 20 seconds.
  • To stop: pull the plug. The storage is read-only; nothing gets corrupted.
  • To move it: it's a Pi, a camera and a cable. Pack it up, set it up somewhere else, plug it in. No reconfiguration needed.

7. When things go wrong

The Wi-Fi zeitspiegel doesn't appear after 5 minutes

Plug a USB keyboard into the Pi and look at the console on the HDMI output for error messages — the first-boot script logs there. The most common cause is a country code mismatch blocking the radio; re-bake the card with WIFI_COUNTRY=YourTwoLetterCode make sd.

zeitspiegel.local doesn't resolve

Some phones and routers don't do mDNS. Use the fallback IP http://10.42.0.1 — this always works when you're on the appliance's Wi-Fi.

The mirror looks fine but the delay slider does nothing

Make sure you're connected to the zeitspiegel Wi-Fi (not your home or studio Wi-Fi) when you open the web page. The appliance is only reachable on its own network.

I need to change a setting (camera mode, buffer length, etc.)

ssh zeitspiegel@zeitspiegel.local (uses the SSH key you baked in). The root filesystem is read-only by design, so you need to unseal it for one boot to make changes — the procedure is in deploy/PROVISIONING.md §5.

How do I update to a new version?

Easiest: git pull in your local checkout, run make sd again, swap the card. The Pi treats it as a fresh boot. If you've configured anything custom on the device, back that up first.

For developers

Running Zeitspiegel without a Pi or camera (synthetic source) for development: 

make run-synth
